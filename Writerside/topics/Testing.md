# Testing

In software development, it is common to write tests for implemented functions and other structures.
These tests ensure that the code functions within the specified constraints.
In the Minecraft scene, relying on the possibility of tests is uncommon.
Various reasons contribute to this, but laziness is often a significant factor.

If you decide to contribute to the Microtus project, there is an expectation to write JUnit tests for features, if
possible.
But why should one do this? Minestom is inherently complex, and the connections between individual code components may
not be apparent.
Without tests, changes could inadvertently be merged, negatively impacting the functionality of other elements.

If one has no prior experience with defining such tests, it is advisable to explore this first and, after some practice,
apply it to their feature or bug fix.

Useful links on this topic:

- [Junit 5](https://junit.org/junit5/)
- [Baeldung Junit (Tutorial)](https://www.baeldung.com/junit)
